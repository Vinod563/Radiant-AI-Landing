## Content Writing Patterns

### Partners Page Headline Formula
```
"[Company] Partners — Trusted [Category 1], [Category 2] & [Category 3] Network"
```
- "Trusted" as the anchor word
- Three partner categories for breadth
- "Network" implies interconnected value, not just a list

### Meta Description Pattern
```
"Accelerate [outcome] with [Company]'s ecosystem of trusted partners. Explore [type 1], [type 2], and [type 3] delivering [value proposition]."
```
- "Accelerate" — speed and momentum
- "Ecosystem" — interconnected, not siloed
- "Delivering integrated solutions for growth" — joint value prop

### Partner Categories (Based on Onixnet)
- **Cloud leaders:** Major platform partners (Google Cloud, AWS, Azure)
- **ISVs:** Independent software vendors with complementary solutions
- **Tech innovators:** Emerging technology partners

### Content Structure
1. **Hero:** "Trusted partner network" headline
2. **Partner categories:** Tabbed or segmented display
3. **Per-partner cards:** Logo + tier + brief description + "Learn More"
4. **Joint value proposition:** What clients get from the ecosystem
5. **Certification showcase:** Badge grid
6. **CTA:** "Leverage our partner ecosystem"

### Key Takeaway for Radiant
- Partner pages should frame the ecosystem as a client benefit
- Use "ecosystem" not "list" — implies integrated value
- Three categories minimum: platform partners, technology partners, domain partners
- Every partner entry should answer: "What does this mean for our clients?"
- Certification badges are visual trust anchors — display prominently
- The meta description should start with an action verb (Accelerate, Leverage, Unlock)
